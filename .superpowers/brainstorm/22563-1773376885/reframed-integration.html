ChangeTracks + Proof: Reframed Integration

What ChangeTracks actually provides that Proof doesn't have

Reframe: ChangeTracks doesn't replace Proof's review UI or accept/reject machinery. Proof owns the editing surface and will build their own review workflows. ChangeTracks provides the deliberation layer and portable format that Proof's in-memory CRDT state doesn't capture.

What Proof Has vs. What ChangeTracks Adds

Proof Has: Suggestions

proofSuggestion marks (insert/delete/replace) live in ProseMirror state and Y.Doc. They render inline with accept/reject affordances. This is their track-changes.

Proof owns this

Proof Lacks: Reasoning

When an agent makes a suggestion, there's no structured place for why. No threaded discussion on a change. No amendment history showing how a proposal evolved.

Gap

Proof Has: Provenance

proofAuthored marks track human vs AI at the character level. Heatmap gutter shows authorship density. This is unique to Proof.

Proof owns this

Proof Lacks: File Portability

All change history lives in Y.Doc binary state and server-side marks. Export to .md loses everything. No way to email someone a markdown file with tracked changes visible.

Gap

Proof Has: Real-time Collab

Yjs + Hocuspocus gives live multi-cursor editing. ChangeTracks is single-user. No overlap here — Proof owns this completely.

Proof owns this

Proof Lacks: Agent Enforcement

Agents can do raw edits via HTTP bridge. No hook system to intercept and force agents through tracked change proposals. The rewrite barrier is a blunt on/off.

Gap

Proof Has: Server-Side Marks

Marks stored independently from doc content with quote-based anchoring. Supports comments, approvals, flags. But marks are ephemeral server state, not in the file.

Proof owns this

Proof Lacks: Deliberation History

No structured format for: "Agent A proposed X because Y. Agent B superseded with Z because W. Human approved with note." The conversation about changes has no home.

Gap

The pitch: Proof owns the editing experience and real-time collaboration. ChangeTracks owns the memory — the structured record of why changes were made, who deliberated, and what the reasoning was. That memory persists in the file itself, portable across any editor, readable by any LLM, diffable in any git tool.
